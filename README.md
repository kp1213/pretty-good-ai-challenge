# Pretty Good AI Engineering Challenge

An automated AI-powered patient simulator that places outbound phone calls to a healthcare AI phone agent, conducts realistic voice conversations, records each interaction, generates transcripts, and produces structured bug reports for quality assurance testing.

---

# Project Overview

This project was developed as part of the Pretty Good AI Engineering Challenge.

The objective was to build an automated patient simulator capable of interacting with a healthcare AI phone agent over the phone. The system places outbound calls, conducts realistic patient conversations using OpenAI, records each conversation, generates transcripts, and documents any bugs or unexpected behaviors observed during testing.

To evaluate the healthcare AI across a variety of real-world situations, I created ten unique patient scenarios covering appointment scheduling, medication refills, emergency symptoms, follow-up care, appointment rescheduling, new patient registration, administrative questions, and acute illness. Each completed call produced a recording, transcript, and structured bug report for later review.

# Running the Project

The project uses **three terminal windows** during execution.

## Terminal 1 – Start the Flask Server

```bash
python src/server.py
```

This starts the Flask application that receives Twilio voice webhooks.

---

## Terminal 2 – Start Ngrok

```bash
ngrok http 5000
```

Copy the HTTPS forwarding URL generated by Ngrok and update the webhook URL inside:

```text
src/caller.py
```

---

## Terminal 3 – Start the Patient Simulator

```bash
python src/main.py
```

This initiates the outbound phone call to the healthcare AI phone agent.

---

Once the call begins, the application automatically:

- Places the outbound phone call
- Conducts the AI-to-AI conversation
- Records the audio
- Generates a transcript
- Generates a structured bug report

# Testing Methodology

Each official scenario followed the same workflow:

1. Configure the patient profile in `prompts.py`
2. Start the Flask server
3. Expose the local server with Ngrok
4. Place an outbound call using Twilio
5. Conduct a complete AI-to-AI conversation
6. Save the audio recording
7. Generate a transcript
8. Review the conversation
9. Document any bugs or unexpected behavior

Each completed scenario produced:

- Audio recording (.mp3)
- Conversation transcript (.txt)
- Structured bug report (.md)

# Results

The project successfully completed ten automated healthcare conversations spanning multiple healthcare workflows.

Testing revealed several recurring behaviors and potential issues including:

- Repeated identity verification requests
- Repeated phone number verification
- Incomplete workflow progression after patient verification
- Unsuccessful representative transfer workflow
- Speech recognition inconsistencies
- Different handling of emergency versus non-emergency scenarios

These observations were documented in structured bug reports for each official test scenario.
